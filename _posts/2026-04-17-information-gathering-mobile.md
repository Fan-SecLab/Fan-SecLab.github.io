# 移动端信息收集（入门向）

## 一、APP 逆向工程

### 1.1 起源

一款 APP（抖音、快手之类的）或者一个电脑软件，都是由开发语言编写出来的。开发完成之后，想要做成可供用户下载安装的软件，需要对程序代码和运行环境等进行**编译和打包**。而我们下载安装 APP 到手机或电脑上的过程，其实就是**解包**——软件的代码程序就解压到了你的设备上。

当你双击软件或点击 APP 时，功能就能用了，因为解压到设备上的程序运行起来了。那么 APP 的代码已经在你的设备上了，你就可以想办法：

- 修改这些代码以达到某些目的
- 看看代码里是否有你需要的信息

但这些代码经过编译，甚至有些 APP 会**对代码进行加密和混淆**，让你不能轻松看出内容。逆向工程的工作就因此出现了。

### 1.2 安卓正向开发流程

```
Java 语法 + 安卓框架
    → 实现功能（Java / Kotlin + C）
    → 编译打包
    → jaden.apk（包含：资源文件(图片、音频等)、代码文件(dex、so 等二进制文件)）
```

### 1.3 逆向的三大方向

以安卓为例，网络攻防中主要涉及以下三个需求：

#### 方向一：逆向破解

```
jaden.apk
  → 反编译（加壳脱壳、混淆反混淆、反汇编）
  → Java / C / Smali / 汇编
  → 修改逻辑
  → Hook 调试
  → 二次打包
```

#### 方向二：逆向分析

```
jaden.apk
  → 反编译（加壳脱壳、混淆反混淆、反汇编）
  → Java / C / Smali / 汇编
  → 分析逻辑 → Hook 验证
  → 自行写代码实现逻辑
  → 通过编写的代码加工请求数据包
  → 发送请求获取数据 → 达到测试目的
```

#### 方向三：逆向木马病毒

进行行为分析和溯源。

> 💡 因为逆向就是将二进制代码逆向回比较可读的代码形式，所以逆向也叫做**二进制逆向**。

### 1.4 安全测试中的应用

如果我们想在 APP 上挖洞，通过常规查询方式可能找不到域名——因为域名都在 APP 里面。这时就需要反编译 APP，提取相关 IP 地址，并且有可能会发现**大量接口 IP 和内网 IP**，从而获取不少安全漏洞。

## 二、微信公众号和小程序抓包

### 2.1 基本介绍

现在很多企业都会做自己的微信公众号，这些公众号里面其实能找到很多的域名信息和漏洞。公众号一般用手机访问，那么如何获取这些信息呢？这就需要借助**抓包工具**了。

抓包工具能够帮我们抓取网络请求的数据包。不管是 APP 还是网站，很多功能都需要网络交互来完成——比如打开抖音的瞬间，APP 向后台发送了网络请求获取视频数据。这时我们就可以通过抓包工具抓取请求，请求中会有域名地址信息，往往会有意外收获。

### 2.2 所需工具

#### Burp Suite（BP）

需要自己配置证书。

#### Proxifier

Proxifier 是一款功能强大的代理客户端，它能够让不支持代理功能的程序通过代理服务器访问网络。支持多种操作系统和代理协议，允许用户基于程序和端口的特定需求灵活配置。

**主要用途**：通过代理服务器强制所有网络请求走代理，对不支持代理的程序实现网络代理功能。

### 2.3 Proxifier 配置步骤

**步骤 1**：点击 `Profile` → 选择 `Proxy Servers`，设置监听同 Burp 代理端口（`127.0.0.1:8888`）。

![Proxifier 代理设置](/assets/images/03/image.png)

**步骤 2**：`Applications` 先不选。先打开一个微信小程序，然后通过任务管理器找到程序存放的目录，从 Applications 里选择需要的程序——找到 `wechatappex.exe`，点击 OK。

![选择 wechatappex.exe](/assets/images/03/image-1.png)

![选择程序路径](/assets/images/03/image-2.png)

**步骤 3**：选择 `Advanced`，激活 **HTTP Proxy Servers Support** 和 **Handle Direct Connections**，到这里就可以愉快地抓包了。

![Advanced 设置](/assets/images/03/image-3.png)

## 三、溯源信息收集

### 3.1 作用

**目标**：发现特定攻击方的相关信息，通过获得相关信息定位到某个具体的个人。

**溯源需要收集的信息**：

| 信息项 | 说明 |
|:---|:---|
| 类型 | 攻击类型分类 |
| 恶意文件 | 攻击使用的恶意样本 |
| 受攻击域名 / IP | 被攻击的目标 |
| 攻击时间 | 攻击发生的时间节点 |
| 攻击 IP | 攻击来源 IP |
| 预警平台 | 发现攻击的监测平台 |

### 3.3 溯源流程

1. **收集信息**——获取攻击相关数据
2. **判断目标 IP 是否存在溯源价值**
   - ✅ VPS、傀儡机 → 需要溯源
   - ❌ 移动热点 → 无需溯源
3. **反制手段**：反打、反向渗透

### 3.4 总结

根据攻击 IP，可以查找以下信息链：

```
攻击 IP
  → 查找定位
  → IP 反查域名
  → 备案号 → 备案信息 → 注册人
  → 邮箱、电话、姓名
  → QQ、微信、微博、支付宝
  → 曾经用过的密码
  → ……（很多信息从社工库 SGK 查询）
```
