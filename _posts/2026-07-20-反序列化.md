# 反序列化漏洞

---

## 序列化/反序列化介绍

### 序列化

把内存中的对象，转换成可存储、可传输的字符串的过程。

> 💡 Java User 对象、Python 字典、C++ 结构体 → 变成一串字节

### 反序列化

把磁盘 / 网络传来的字节 / 字符串，还原回程序内存里可用的对象。

### 为什么需要序列化

| 场景 | 说明 |
|------|------|
| 持久化存储 | 对象只在内存存活，程序关闭、电脑断电就丢失。序列化后保存到本地磁盘。 |
| 网络传输 | 网络只能传二进制字节，不能直接发送内存对象 |
| 跨语言数据交换 | 不同语言内存结构完全不一样，统一序列化格式作为中间载体，实现互通。 |

> 💡 序列化是将变量或对象转换成字符串的过程。方法：`serialize()`
>
> 反序列化是将字符串转换成变量或对象的过程。方法：`unserialize()`

---

## 反序列化漏洞原理

`serialize()` 和 `unserialize()` 在 PHP 内部实现上是没有漏洞的，之所以会产生反序列化漏洞是因为应用程序在处理对象、魔术函数以及序列化相关问题的时候导致的。

当传给 `unserialize()` 的参数可控时，那么用户就可以注入精心构造的 payload。当进行反序列化的时候就有可能会触发对象中的一些魔术方法，造成意想不到的危害。

> ⚠️ **出现这个漏洞有两个条件：**
> 1. 参数可控
> 2. 类中使用了魔术方法
>
> 另外，反序列化漏洞其实在没有源代码的情况下是不好挖的，因为并不知道别人的类怎么写的，所以无法构造相应的 payload，所以基本上都是在代码审计的时候挖这种漏洞。但是有些大家都使用的 web 框架或者 web 服务应用程序，他们的代码是开源的，所以大家可以挖这些代码的反序列化漏洞，而且好多漏洞别人都挖好了，直接利用就行。

### 序列化演示

```php
<?php 
class S {                    // 创建一个类
    public $test = "pikachu"; // 创建一个公有变量
}

$s = new S();               // 实例化一个对象
echo serialize($s);         // 把这个对象序列化并输出
?>
```

序列化后得到的结果：

```
O:1:"S":1:{s:4:"test";s:7:"pikachu";}
```

| 符号 | 含义 |
|------|------|
| `O` | 代表 object |
| `1` | 对象名字长度为一个字符 |
| `S` | 对象的名称 |
| `1` | 对象里面有一个变量 |
| `s` | 数据类型：字符串 |
| `4` | 变量名称的长度 |
| `test` | 变量名称 |
| `s` | 数据类型：字符串 |
| `7` | 变量值的长度 |
| `pikachu` | 变量值 |

### 反序列化演示

```php
<?php 
class S {                    // 创建一个类
    public $test = "pikachu"; // 创建一个公有变量
}

$s = new S();               // 实例化一个对象
$a = serialize($s);         // 序列化
$b = unserialize($a);       // 反序列化

var_dump($b);               // 输出反序列化后的对象结构
?>
```

反序列化的结果：

```
object(S)#2 (1) {["test"]=>string(7) "pikachu"}
```

| 符号 | 含义 |
|------|------|
| `object` | 代表这是一个对象类型，不是数组、字符串、数字 |
| `(S)` | 这个对象所属的类名是 S |
| `#2` | 第 2 个实例对象 |
| `(1)` | 变量个数 |
| `test` | 属性名 |
| `string(7)` | 字符串长度 |
| `pikachu` | 属性值 |

> ⚠️ 序列化和反序列化本身没有问题，但是如果反序列化的内容是用户可以控制的，且后台不正当的使用了 PHP 中的魔法函数，就会导致安全问题。

### 常见的几个魔法函数

| 魔法函数 | 触发时机 |
|----------|----------|
| `__construct()` | 当一个对象创建时被调用 |
| `__destruct()` | 当一个对象销毁时被调用 |
| `__toString()` | 当一个对象被当作一个字符串使用 |
| `__sleep()` | 在对象在被序列化之前运行 |
| `__wakeup()` | 将在反序列化之前立即被调用 |

## pikachu 漏洞示例

源代码：

```php
<?php
class S {                        // 创建一个类
    var $test = "pikachu";       // 默认属性值

    function __construct() {     // 创建对象时自动调用
        echo $this->test;        // 打印 test 属性值
    }
}

if (isset($_POST['o'])) {
    $s = $_POST['o'];            // 接收用户输入
    
    if (!@$unser = unserialize($s)) {  // 反序列化失败
        $html .= "<p>大兄弟,来点劲爆点儿的!</p>";
    } else {                     // 反序列化成功
        // 直接取 test 属性拼进 HTML，无任何过滤转义 → XSS 漏洞
        $html .= "<p>{$unser->test}</p>";
    }
}
?>
```

payload：

```
O:1:"S":1:{s:4:"test";s:29:"<script>alert('xss')</script>";}
```

> 💡 这个 payload 提交上去，被反序列化成功。`unserialize()` 的行为就是重建对象；重建时完全抛弃类内属性默认值，使用序列化内记录的值。

```php
// 实际执行效果等同于：
$unser->test = "<script>alert('xss')</script>";
```

直接拼进 HTML 标签，进行弹窗。

![XSS弹窗效果](/assets/images/16/image-20260720164156520.png)

> ⚠️ 出现这个漏洞有两个条件：
> 1. 参数可控（参数数据可以自己指定）
> 2. 类中使用了魔术方法（魔术方法是进行反序列化的时候自动执行的方法）

## 私有变量和公有变量

```php
<?php
header("content-type: text/html; charset=utf-8");

class person {
    public $name = "Jaden is good";  // 公有变量 — 类内外均可访问
    private $age = 18;               // 私有变量 — 仅类内可访问

    public function jaden() {        // 公有方法
        echo $this->name . '<br>';
        echo $this->age . '<br>';
        $this->jaden2() . '<br>';    // 类内调用私有方法 OK
    }

    private function jaden2() {      // 私有方法 — 仅类内可调用
        echo 'no good!';
    }
}
?>
```

| 访问修饰符 | 含义 | 示例 |
|-----------|------|------|
| `public` | 可以类内外调用 | `$a->name` ✅ |
| `private` | 只能类内调用 | `$a->age` ❌ |

```php
$a = new person();           // 实例化
echo $a->name . '<br>';     // name 是公有变量 → 成功
echo $a->age;               // age 是私有变量 → error
$a->jaden();                // jaden 是公有函数 → 成功
$a->jaden2();               // jaden2 是私有函数 → error
```

## 类和对象

### 类

一组具有相似特征和行为的对象的抽象描述。它定义了对象所具有的属性（成员变量）和行为（方法），是创建对象的模板或蓝图。类是抽象的，不占用内存空间，它只是对对象的描述。

```php
class car {
    // 属性
    public $品牌;
    public $型号;
    public $颜色;

    // 方法
    public function 启动() {
        echo "汽车启动了！";
    }

    public function 加速() {
        echo "汽车加速中...";
    }

    public function 停止() {
        echo "汽车停止了。";
    }
}
```

### 对象

类的实例化结果，是类的具体实体。它具有类所定义的属性和行为，并且可以在程序中被直接使用。对象是具体的，占用内存空间。

```php
// 创建汽车对象
$奔驰迈巴赫 = new car();

// 设置属性值
$奔驰迈巴赫->品牌 = "奔驰";
$奔驰迈巴赫->型号 = "迈巴赫";
$奔驰迈巴赫->颜色 = "黑色";

// 调用方法
$奔驰迈巴赫->启动();
$奔驰迈巴赫->加速();
$奔驰迈巴赫->停止();
```

## 魔术方法

| 魔术方法 | 触发时机 |
|----------|----------|
| `__construct()` | 当实例化一个类的对象时，自动触发 |
| `__destruct()` | 销毁一个类的对象时自动触发 |
| `__sleep()` | 执行 `serialize()` 前如果存在 `__sleep()` 方法，会自动触发 |
| `__wakeup()` | 执行 `unserialize()` 前，如果存在 `__wakeup()` 方法，自动触发。与 `__sleep()` 相对，但不能和 `sleep` 同时使用在一个类中，不然 `wakeup` 方法不会执行 |
| `__invoke()` | 当尝试以调用函数的方式调用一个对象时，自动调用。本特性只在 PHP 5.3.0 及以上版本有效 |
| `__toString()` | 类的对象被用作字符串输出时，自动触发 |
| `__get()` | 访问类中私有变量或者不存在的变量时，自动触发执行 |

### `__sleep()`

```php
<?php
header("content-type: text/html; charset=utf-8");

class S {
    var $test = "pikachu";

    function __construct() {
        echo "construct" . "</br>";
    }

    function __destruct() {
        echo "destruct" . "</br>";
    }

    function __sleep() {        // 序列化前自动触发
        echo "sleep" . "</br>";
    }

```

<img src="/assets/images/16/image-20260720224458955.png" alt="sleep执行结果" style="zoom:33%;" />

> 💡 `destruct` 之所以在最后是因为在执行 `__sleep` 函数时，仍在调用类，调用完毕后才会自动销毁。

### `__wakeup()`

```php
<?php
header("content-type: text/html; charset=utf-8");

class S {
    var $test = "pikachu";

    function __construct() {
        echo "construct" . "</br>";
    }

    function __destruct() {
        echo "destruct" . "</br>";
    }

    function __wakeup() {       // 反序列化前自动触发
        echo "wakeup" . "</br>";
    }
}

$a = new S();
$b = serialize($a);
unserialize($b);  // 反序列化 → 自动触发 __wakeup()
?>
```

<img src="/assets/images/16/image-20260720225004506.png" alt="wakeup执行结果" style="zoom:33%;" />

执行 `unserialize()` 前，如果存在 `__wakeup()` 方法，自动触发。与 `__sleep()` 相对，但是不能和 `sleep` 同时使用在一个类中，不然 `wakeup` 方法不会执行。

> 💡 为什么出现 2 个 destruct？因为销毁了 `$a`、`$b` 这 2 个对象。

### `__invoke()`

```php
<?php
header("content-type: text/html; charset=utf-8");

class S {
    var $test = "pikachu";

    function __construct() {
        echo "construct" . "</br>";
    }

    function __destruct() {
        echo "destruct" . "</br>";
    }

    function __wakeup() {
        echo "wakeup" . "</br>";
    }

    function __invoke() {       // 对象被当作函数调用时自动触发
        echo "invoke" . "</br>";
    }
}

$a = new S();
$b = serialize($a);
unserialize($b);
$a();  // 把对象当函数调用 → 触发 __invoke()
?>
```

<img src="/assets/images/16/image-20260720230416162.png" alt="invoke执行结果" style="zoom:33%;" />

当尝试以调用函数的方式调用一个对象时（例如把类当函数去调用 `$a();`），`__invoke()` 方法会被自动调用。本特性只在 PHP 5.3.0 及以上版本有效。

### `__get()`

```php
<?php
header("content-type: text/html; charset=utf-8");

class S {
    var $test = "pikachu";
    private $age = 18;          // 私有变量

    function __construct() {
        echo "construct" . "</br>";
    }

    function __destruct() {
        echo "destruct" . "</br>";
    }

    function __wakeup() {
        echo "wakeup" . "</br>";
    }

    function __invoke() {
        echo "invoke" . "</br>";
    }

    function __toString() {
        echo "toString" . "</br>";
        return "";
    }

    function __get($k) {        // 访问私有/不存在属性时自动触发
        // $k 接收属性名，$this->$k 获取实际值
        echo "get" . "</br>" . $k . "</br>" . $this->$k . "</br>";
    }
}

$a = new S();
$b = serialize($a);
unserialize($b);
$a();           // 触发 __invoke()
echo $a;        // 触发 __toString()
$a->age;        // 访问私有属性 → 触发 __get('age')
?>
```

<img src="/assets/images/16/image-20260720232022718.png" alt="get执行结果" style="zoom:33%;" />

访问类中私有变量时，自动触发执行。访问类中私有变量或者不存在的变量时，自动触发执行。

> 💡 外部 `$a` 读取私有属性 `age`，触发 `__get($k)`，`$k` 接收字符串 `"age"`。

## 反序列化漏洞触发 RCE

```php
<?php
header("content-type: text/html; charset=utf-8");

class person {
    public $name = "echo 'Jaden is good'";  // 默认值

    public function __wakeup() {            // 反序列化时自动触发
        eval("$this->name;");               // ⚠️ 危险：将属性值作为 PHP 代码执行
    }
}

if (isset($_GET['arg'])) {
    $arg = $_GET['arg'];
    unserialize($arg);          // 反序列化用户输入 → 触发 __wakeup()
} else {
    echo "<h1>what?no arg??<h1/>";
}
?>
```

payload：

```
O:6:"person":1:{s:4:"name";s:9:"phpinfo()";}
```

### 攻击流程

| 步骤 | 说明 |
|------|------|
| 1. 传入 payload | 用户通过 GET 参数 `arg` 传入恶意序列化字符串 |
| 2. 触发 `__wakeup()` | 执行 `unserialize()` 前自动触发 |
| 3. 调用 `$this->name` | 取出 `name` 属性的值 |
| 4. `eval()` 执行 | 把括号里面的语句当成 PHP 代码执行 |

> 🎯 当你执行反序列化时 `unserialize($arg)`，`unserialize()` 的行为就是重建对象，重建时完全抛弃类内属性默认值，变成使用序列化内记录的值。然后被 `eval` 函数当成 PHP 代码执行，就出现了 `phpinfo()`。

## POP 链构造

POP 链是反序列化漏洞利用中的一种常用方法，即寻找程序环境中已经定义或能够动态加载的对象中的属性或函数，将一些能够被调用的函数组合起来，达到目的的操作。

> 💡 POP 又称之为面向属性编程（Property Oriented Programming），常用于上层语言构造特定调用链的方法，与二进制利用中的面向返回编程（Return-Oriented Programming）的原理相似，都是从现有运行环境中寻找一系列的代码或者指令调用，然后根据需求构成一组连续的调用链，最终达到攻击者邪恶的目的。

```php
<?php
header("content-type: text/html; charset=utf-8");

class Start
{
    public $name = 'guest';
    public $flag = 'system("cat 127.0.0.1/etc/hint");';

    public function __construct() {
        // 当创建完一个类时，自动执行
        echo "hi man what are you doing!";
    }

    public function _sayhello() {
        echo '>>>>>>>>>>' . $this->name;
        return 'ok';
    }

    public function __wakeup() {
        // 反序列化时自动触发 → 调用 _sayhello()
        echo "file_data-->" . '<br>';
        $this->_sayhello();
    }

    public function __get($cc) {
        // 访问私有/不存在属性时自动触发
        echo "give you flag : " . $this->flag;
        return;
    }
}

class Info
{
    private $phonenumber = 666666;
    public $promise = 'I do';

    public function __construct() {
        // 实例化时自动触发
        $this->promise = 'I will not !!!!';
        return $this->promise;
    }

    public function __toString() {
        // 对象被当作字符串时触发 → 访问不存在的 jaden 属性
        return $this->file['filename']->jaden;
        // $this->file['filename'] 是 Room 对象
        // → 访问 ->jaden（不存在）→ 触发 Room::__get()
    }
}

class Room
{
    public $filename = 'c:\\';
    public $sth_to_set;
    public $a = '';

    public function __get($name) {
        // 访问私有/不存在属性时自动触发
        $function = $this->a;       // $this->a 是 Room 实例
        $k = $function();           // 把 Room 实例当函数调用 → 触发 __invoke()
        echo '<br>' . '00000' . $k . '<br>';
        return $k;
    }

    public function Get_hint($file) {
        // 🎯 目标函数：读取服务器文件并 base64 回显
        $hint = base64_encode(file_get_contents($file));
        echo $hint;
        return 'jadenok';
    }

    public function __invoke() {
        // 对象被当作函数调用时触发 → 调用 Get_hint()
        $content = $this->Get_hint($this->filename);
        echo '>>>' . $content . '<br>';
        return 'jaden111';
    }
}

if (isset($_GET['jaden'])) {
    unserialize($_GET['jaden']);    // 入口：反序列化用户输入
} else {
    $hi = new Start();              // 无参数时正常实例化
}
?>
```

### 代码分析

直接访问：

<img src="/assets/images/16/image-20260721223145094.png" alt="直接访问" style="zoom:33%;" />

客户端由于没有传递参数，服务器端实例化了一个 `Start` 类，而实例化 `Start` 类触发 `__construct` 魔术方法，该方法输出了一句话，至此，结束。

---

### POP 链逐步分析

> 🎯 **目标**：利用 `Room` 类的 `Get_hint()` 方法中的 `file_get_contents()` 读取服务器文件并通过 `echo()` 回显。

**步骤 1：找到敏感函数**

`class Room` 是调用链的末端，主要是想利用一下敏感函数 `file_get_contents()` 可以读取服务器上的文件并通过 `echo()` 回显到客户端。需要执行一下 `class Room` 类中的 `Get_hint()` 方法。

**步骤 2：利用 `__invoke()` 调用 `Get_hint()`**

`__invoke()` 魔术方法可以实现调用 `Get_hint()`，触发条件是实例化的 `class Room` 类对象被当作**函数调用**。观察代码发现 `__get()` 魔术方法可以实现"实例化的类以函数方式调用"这个条件 `$k = $function()`，只要我们想办法给 `$a` 重新赋值一个实例化的 `Room` 类的对象，就可以了。

> 💡 怎么给 `$a` 的值赋值为 `Room` 类的对象？在写 payload 时可以借助 `__construct()` 方法来进行属性重新赋值。

**步骤 3：触发 `__get()` 方法**

`__get()` 方法的触发条件是访问实例化 `Room` 类的私有变量或者不存在的变量。我们发现 `Info` 类中的 `__toString()` 方法进行了连续调用 `$this->file['filename']->jaden`，很明显不存在 `jaden` 这个属性，这个就可以触发 `Room` 类中的 `__get()` 方法。

> 💡 我们想办法将 `file['filename']` 的值赋值为一个 `Room` 类实例化对象就可以了。当这个对象再次调用 `jaden` 属性时发现 `jaden` 属性不存在 `Room` 类中，就会当作私有属性来调用，从而触发 `Room` 类中的 `__get()` 方法。

**步骤 4：赋值 `file['filename']`**

怎么给 `Info` 类的 `file['filename']` 的值赋值为 `Room` 类的实例化对象呢？在写 payload 时可以借助 `__construct()` 方法来进行属性重新赋值。

**步骤 5：触发 `__toString()` 方法**

`__toString()` 方法的运行需要实例化的 `class Info` 对象被当作字符串输出。正好我们发现 `Start` 类中有个 `_sayhello()` 方法，里面有个代码是 `echo $this->name`，通过 `echo` 打印字符串。

> 💡 在写 payload 时我们想办法将 `Start` 类中的 `name` 属性借助 `__construct()` 方法来进行属性重新赋值，赋值为 `Info` 类的实例化对象即可。

**步骤 6：触发 `_sayhello()` 方法**

我们发现 `Start` 类中的 `__wakeup()` 方法里面调用了 `_sayhello()` 方法，所以想办法让 `__wakeup` 方法自动执行即可。而 `__wakeup` 方法的执行需要对 `Start` 类的实例化对象的序列化的字符串进行反序列化的时候触发，那么正好访问入口的地方有反序列化动作。

disable_functions = system,passthru,shell_exec,exec,popen

## 总结

| 漏洞类型 | 核心条件 | 危害 | 防护思路 |
|----------|----------|------|----------|
| PHP 反序列化 | 参数可控 + 魔术方法 | XSS / RCE / 任意文件读取 | `disable_functions`、控制传入参数、避免魔术方法中执行敏感操作 |
| JBoss 反序列化 | Java 反序列化接口暴露 | 远程命令执行 | 升级版本、限制 `/invoker` 访问 |
| WebLogic 反序列化 | T3 协议 / IIOP 协议反序列化 | 远程命令执行 | 升级补丁、配置黑名单、限制端口访问 |

> 💡 **核心要点回顾：**
> - 序列化本身不是漏洞，漏洞来源于 **参数可控 + 魔术方法利用**
> - POP 链的本质是**从入口到目标函数的属性/方法调用链路**
> - Java 中间件（JBoss、WebLogic）的反序列化漏洞通常是框架层面的，利用工具相对成熟
> - 代码审计是发现反序列化漏洞的主要手段
