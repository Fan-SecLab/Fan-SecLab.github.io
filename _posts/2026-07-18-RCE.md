# 命令执行漏洞

命令执行漏洞，英文 **remote command/code execute**，简称 RCE，包含远程命令执行和远程代码执行。

## 远程系统命令执行

一般出现这种漏洞，是因为应用系统从设计上需要给用户提供指定的远程命令操作的接口，比如我们常见的路由器、防火墙、入侵检测等设备的 web 管理界面上一般会给用户提供一个 ping 操作的 web 界面，用户从 web 界面输入目标 IP，提交后，后台会对该 IP 地址进行一次 ping 测试，并返回测试结果。而如果设计者在完成该功能时，没有做严格的安全控制，则可能会导致攻击者通过该接口提交"意想不到"的命令，从而让后台进行执行，从而控制整个后台服务器。

### windows示例

我们打开 cmd 窗口，这里面也可以执行 ping 指令：

```cmd
ping www.baidu.com
```

<img src="/assets/images/15/image-20260718152855720.png" alt="image-20260718152855720" style="zoom:33%;" />

但是如果在 ping 指令后加上例如 ipconfig 的一些指令，那么你的信息就会暴露：

```cmd
ping www.baidu.com&&ipconfig
```

> 💡 `&&`：前面命令执行成功后，继续执行后面的命令。

<img src="/assets/images/15/image-20260718153118316.png" alt="image-20260718153118316" style="zoom:33%;" />

```cmd

```

> 💡 `||`：前面命令执行失败（报错）后，执行后面的命令。由于前面 ping 不通，所以执行了 ipconfig。

![image-20260718153354924](/assets/images/15/image-20260718153354924.png)

```cmd
ping www.baidu.com|ipconfig
```

> 💡 `|` 管道符，这里只执行后面的命令。

<img src="/assets/images/15/image-20260718160643072.png" alt="image-20260718160643072" style="zoom:33%;" />

源代码：

![image-20260718161438225](/assets/images/15/image-20260718161438225.png)

```php
// 判断 POST 请求中是否存在 submit 参数，且 ipaddress 字段不为空
if(isset($_POST['submit']) && $_POST['ipaddress']!=null){

    // php_uname('s')：获取当前服务器操作系统名称（Windows/Linux）
    // stristr(a,b)：不区分大小写查找字符串 b 是否存在于 a 中
    // 整体：判断服务器是否为 Windows 系统
    if(stristr(php_uname('s'), 'windows')){

        // shell_exec()：PHP 执行系统 shell 命令，返回命令输出结果 —— 高危函数！
        // 直接把用户可控变量拼入系统命令
        shell_exec('ping '.$ip);
    }
}
```

`php_uname` — 返回运行 PHP 的系统的有关信息。

原型：`string php_uname ([ string $mode = "a" ] )`

| 参数 mode | 含义 | 示例 |
|-----------|------|------|
| `'a'` | 默认，包含序列 "s n r v m" 里所有模式 | — |
| `'s'` | 操作系统名称 | FreeBSD |
| `'n'` | 主机名 | localhost.example.com |
| `'r'` | 版本名称 | 5.1.2-RELEASE |
| `'v'` | 版本信息 | 因系统而异 |
| `'m'` | 机器类型 | i386 |

`stristr(a,b)`：不区分大小写查找字符串 b 是否存在于 a 中。整体是判断服务器是否为 Windows 系统。

`shell_exec()`：PHP 执行系统 shell 命令，返回命令输出结果，**高危函数**。`'ping '.$ip`，直接把用户可控变量拼入系统命令，没做处理，然后当成命令执行。

## 远程代码执行漏洞

同样的道理，因为需求设计，后台有时候也会把用户的输入作为代码的一部分进行执行，也就造成了远程代码执行漏洞。因此，如果需要给前端用户提供操作类的 API 接口，一定需要对接口输入的内容进行严格的判断，比如实施严格的白名单。

### php中执行代码的方法

代码执行与加密相关函数：`eval`、`assert`、`call_user_func`、`base64_decode`、`gzinflate`、`gzuncompress`、`gzdecode`、`str_rot13`。

### pikachu示例

RCE-exec "eval"：

![image-20260718164224349](/assets/images/15/image-20260718164224349.png)

当我们输入：

```
phpinfo();
```

![image-20260718164353743](/assets/images/15/image-20260718164353743.png)

输入的东西被当成代码执行，说明存在远程代码执行漏洞。

### 源代码

![image-20260718164539109](/assets/images/15/image-20260718164539109.png)

```php
// 判断 POST 请求是否携带 submit 参数，且 txt 参数不为空
if(isset($_POST['submit']) && $_POST['txt'] != null){

    // eval()：把传入的字符串当作 PHP 代码直接执行 —— 高危函数！
    // @ 抑制错误输出
    // !eval() 取反：若 eval 执行返回 false，进入 if 分支输出提示文字
    if(@!eval($_POST['txt'])){

    }
}
```

> ⚠️ `eval($_POST['txt'])` 这个就相当于一句话木马，密码就是 `txt`。

然后我们用蚁剑来连接，因为这个一句话木马是在这个 PHP 文件中的，我们直接把 pikachu 的 URL 提交上去：

![image-20260718165422895](/assets/images/15/image-20260718165422895.png)

发现连接不成，这是因为 `if(isset($_POST['submit']) && $_POST['txt'] != null)` 这里要求要提交 2 个参数，一个 `submit`，一个 `txt`。我们直接空提交，抓包来看看。

![image-20260718165927651](/assets/images/15/image-20260718165927651.png)

`%E6%8F%90%E4%BA%A4`：URL 解码就是「提交」。

所以用蚁剑进行连接的时候要在请求体（HTTP BODY）加上 `submit=提交`，因为 txt 是空，所以不用写：

<img src="/assets/images/15/image-20260718170342171.png" alt="image-20260718170342171" style="zoom:33%;" />

然后就测试成功了，获取到了对方主机：

### impossible等级

源代码：

![image-20260718210817981](/assets/images/15/image-20260718210817981.png)

| 函数 | 作用 |
|------|------|
| `stripslashes()` | 删除字符串里转义反斜杠 `\`，把转义后的字符还原成原始字符 |
| `explode(分隔符, 字符串)` | 以指定字符 `.` 分割字符串，返回数组 |
| `is_numeric(变量)` | 判断变量是否为数字 |
| `sizeof()` | 统计数组 / 对象的元素个数 |

在 API 接口输入 `127.0.0.1` 点击提交：

1. `stripslashes()` 先删除转义符
2. `explode(".", "127.0.0.1")` 分割成数组 → `["127", "0", "0", "1"]`
3. `is_numeric(变量)` 判断变量是数字
4. `sizeof()` 确实有 4 个元素
5. 然后再把这 4 个元素用 `.` 拼接回来执行相应的命令

> 🎯 防御总结：对传进来的数据进行过滤检测，不能直接拼接执行命令。或者设置白名单，只让执行规定的几个命令。

## 中间件和框架类的漏洞

开发人员在开发不同网站时，不同网站中肯定有相同的模块，例如登录模块、数据库模块、路由模块。但是如果每次开发不同的网站时都要重新再把这些相同的模块再写一遍，那么就太麻烦了。所以说就出现了框架，框架中内置了很多功能模块，以及相应更加便捷的指令，框架的出现，大大提升了开发人员的开发效率。

框架也算是一种中间件，中间件包括 nginx、httpd、iis、tomcat、PHP-FPM 等。

> ⚠️ 但是如果框架、中间件中如果出现漏洞，那么所有用这些框架搭建的网站就全部也会出现漏洞。

### s2-045漏洞复现

步骤：

1. 搭建环境（和线上真实环境一致），有些公司本身就有测试环境
2. 手工或者工具的形式检测漏洞是否存在，主要是看响应结果来断定是否存在漏洞
   - 手工的方式：抓包修改数据包，添加上攻击者的 payload 看响应结果
   - 工具的方式：直接用别人写好的 PoC 来进行漏洞检测
